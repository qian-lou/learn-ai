# EX-03 · Agent 系统设计面试专项

> **今日目标**：把"设计一个 XX Agent"打成套路题——背熟一张**六维设计 checklist**，掌握 45 分钟白板节奏，能对客服 / 代码 / 深度研究三类高频命题现场展开并主动讲权衡。
> **时长**：~2h ｜ **前置**：建议完成 Day 1–70 主线后学习（重点依赖 Day 30、36、39、49–55 的积累）
> **今日产出**：一张手抄的六维 checklist 卡片 + 一份「客服 agent」完整白板答案（对着墙讲一遍并录音复盘）。

> 对应每日计划「🧩 扩展 / 进阶主题」中的 **Day EX-SD**（P0 应补），不占 Day 1–70 编号。

## 1. 为什么 & 是什么

2026 年 Agent 岗位面试的主观题新常态就是开放式系统设计："给我们客服团队设计一个 agent"。它考的不是你背了多少框架名，而是：**六个固定维度你是否都主动覆盖，且每个维度能给出至少一种权衡**。

给 Java 工程师的类比：这题 ≈ 后端面试里的**"设计一个秒杀系统"**。命题开放，但考点高度固定（秒杀 = 限流/缓存/一致性/降级；Agent = 下面六维）。高分打法完全同构——先澄清约束，再给最小可用架构，然后逐考点主动深入，最后自己把权衡讲出来，不等面试官追问。套路可整体迁移。

### 六维设计 checklist（核心资产，可背）

| 维度 | 必答问题 | 常见失分点 |
|---|---|---|
| ① 任务边界 | agent 做什么、**明确不做什么**？何时拒答/转人工？ | 边界不清，被追问"用户问 X 怎么办"接不住 |
| ② 上下文与记忆 | 单轮上下文怎么组装、预算多少 token？跨会话记什么、存哪？ | 只会说"全塞进 prompt"，毫无预算意识 |
| ③ 工具集与权限 | 有哪些工具？读/写分级？高危操作谁批准？ | 工具不分级，一句"直接调 API"暴露没上过生产 |
| ④ 失败与回退 | 工具挂了/模型幻觉/循环不收敛怎么办？兜底是什么？ | 只画 happy path |
| ⑤ 评估与护栏 | 上线前后各用什么指标度量？注入/越权怎么防？ | 说不出一个可量化指标 |
| ⑥ 成本 | 单次会话约多少 token/多少钱？如何路由与缓存降本？ | 完全没算过账 |

心法：任何 agent 命题，先把这六行在白板角落列出来当"目录"，再逐行填空——面试官会立刻感知到你有工程化方法论，而不是即兴发挥。

## 2. 跟着做（Hands-on）

**Step 1 — 背 45 分钟白板节奏**

| 时间 | 动作 | 话术模板（可直接背） |
|---|---|---|
| 0–5 min | 澄清需求 | "画图之前我先确认三件事：**用户是谁、量级多大、出错的代价有多高**？" |
| 5–15 min | 画主链路 | "我先给一个最小可用版本：入口 → 意图/路由 → 工具或 RAG → 回复 → 兜底。跑通后我们逐块深入。" |
| 15–35 min | 逐维深入 | "接下来我按六个维度过一遍：边界、上下文、工具、失败、评估、成本。"（每维 3–4 分钟；被打断就顺着面试官关心的维度展开） |
| 35–45 min | 主动讲权衡 | "这个设计里我做了三个关键取舍：……如果预算/流量假设变化，我会这样调整……" |

被问倒时的保命话术："这个点我没实际踩过，但基于最小权限/先验证再放量的原则，我会先这样做 ①……，然后用 eval 数据验证再迭代。"——诚实 + 有方法论，远好于硬编。

**Step 2 — 高频命题一：客服 agent（完整示范骨架）**

命题："为一家电商设计一个售后客服 agent。"

*0–5 澄清*（把开放题钉死成有约束的题）：

- 用户是谁？——C 端购物者，中文为主，情绪化输入多。
- 量级？——日均 10 万会话，大促有 5–10 倍峰值 → 成本与路由必须讲。
- 出错代价？——答错运费是小事，**误发退款是资损** → 权限必须分级。
- 范围？——查订单/物流、政策问答、发起退款；营销与法务投诉不做。

*5–15 主链路*（白板文字版）：

```text
用户消息 → 输入护栏(注入/敏感词) → 意图路由(小模型分类)
  ├─ A 政策/FAQ 问答 → RAG(带引用)
  ├─ B 订单/物流查询 → 只读工具 get_order / get_logistics
  └─ C 退款/改地址   → 写工具 + 金额阈值 + 人工审批(HITL)
→ 输出护栏(敏感信息过滤) → 回复
任一支路置信度低 / 用户明确要求 → 转人工
```

*15–35 六维逐个填空*：

- **① 边界**：只做售后域三类任务；维权、法务、议价直接转人工；维护一份显式拒答清单（医疗/法律建议等）。
- **② 上下文与记忆**：单轮 = 系统指令 + 用户画像摘要 + 最近 N 轮对话 + RAG 片段，总预算封顶（如 8k token，超了先裁历史）；跨会话只持久化"用户偏好 + 未决工单"到数据库，按需注入，不常驻 prompt。
- **③ 工具与权限**：读类工具自动执行；写类工具（退款）按金额分级——小额自动、大额人工确认（HITL）；数据层面 agent 只能查**当前会话用户**自己的订单，防横向越权。
- **④ 失败与回退**：工具超时→重试→降级为"结果稍后短信通知"；模型置信度低→转人工；ReAct 循环设步数上限；全链路最外层有统一兜底话术，永不裸抛错误。
- **⑤ 评估与护栏**：离线——意图分类准确率、RAG 引用命中率、注入攻击用例集拦截率；线上——转人工率、一次解决率、CSAT；发布前 eval 回归，线上全链路 tracing。
- **⑥ 成本**：意图分类用小模型，只有复杂支路升级大模型（模型路由）；高频 FAQ 答案缓存；给出单会话成本预算目标并说明怎么算（输入+输出 token × 单价）。

*35–45 主动讲三个权衡*：

1. **全量大模型 vs 路由**：路由可大幅降本，代价是多一层分类错误风险 → 用意图分类 eval 守住阈值再放量。
2. **退款全自动 vs HITL**：全自动省人力但有资损风险 → 按金额阈值分级，是"效率 vs 安全"的滑杆而非二选一。
3. **RAG vs 微调**：售后政策频繁变化 → RAG 更新快、可溯源；微调适合固定话术风格，不适合易变知识。

**Step 3 — 高频命题二 & 三：考点差异（自己练时按此展开）**

| 命题 | 与客服题的主链路差异 | 面试官最爱追的维度 |
|---|---|---|
| **代码 agent**（如内部 code review / 修 bug 助手） | 工具是文件读写 + 命令执行，必须谈**沙箱隔离**；验证闭环天然存在——"测试通过"就是 eval | ③ 权限（执行任意命令的风险）、④ 失败（编辑冲突、改坏回滚）、② 上下文（大仓库怎么检索装配） |
| **深度研究 agent**（给主题产出带引用报告） | 多 agent/多步编排：拆题→并行检索→综合→报告；长任务需**状态持久化 + 断点续跑** | ⑤ 引用溯源与幻觉、⑥ 成本（长链路 token 爆炸）、④ 循环收敛上限 |

注意：这两题你在主线里都真刀真枪做过（代码类思路见 Day 13/15 的工具化经验，研究 agent 就是 Day 41–45 主力项目）——白板上讲的时候**直接引用你项目里的量化数据**（"我做过的版本 p95 从 8s 降到 2.3s"），这是背 checklist 的人给不出的差异化。

## 3. 今日任务

1. **手抄六维 checklist**（不许复制粘贴）：一张卡片，正面六维+必答问题，背面 45 分钟节奏表。目标是脱稿默写。
2. **实战演练客服题**：对着墙（或录音）用 45 分钟节奏完整讲一遍 Step 2 的骨架，允许压缩到 20 分钟；回听录音，检查六维是否全覆盖、是否讲了至少三个权衡。
3. **迁移练习**：按 Step 3 的差异表，给「代码 agent」和「深度研究 agent」各写一页白板骨架（澄清问题清单 + 主链路图 + 六维要点 + 三个权衡），存进你的面试笔记。

**验收标准**：能不看稿在 20 分钟内讲完客服题且六维无遗漏；两份迁移骨架各自的"三个权衡"不与客服题重复。

## 4. 自测清单

- [ ] 我能脱稿默写六维 checklist 及每维的"必答问题"。
- [ ] 我能说出每个维度至少一个真实权衡（而不只是名词）。
- [ ] 我能在 0–5 分钟内问出"用户/量级/错误代价"三个澄清问题，并解释它们分别决定设计的哪一块。
- [ ] 我能讲清客服/代码/研究三类命题的考点差异（权限沙箱 vs 转人工 vs 引用溯源）。
- [ ] 被追问"预算砍半怎么办"，我能现场从路由、缓存、裁上下文、降模型四个抓手给出调整方案。
- [ ] 我能把自己 Day 41–60 项目里的量化数据（eval 准确率、p95、降本比例）自然地嵌进白板讲述。
